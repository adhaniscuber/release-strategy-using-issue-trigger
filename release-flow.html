Workflow Files
create-release-candidate.yml
Buat release/vX.X.x branch dari main
trigger: workflow_dispatch
create-release.yml
Hitung versi, push tag, buat GitHub Release, trigger release.yml
trigger: workflow_dispatch
release.yml
Build image, tentukan deployment chain, auto-dispatch dev atau buka approval issue
trigger: workflow_dispatch ← dari create-release.yml
on-issue-comment.yml
Handle /approve & /reject, validasi approvers.yml, dispatch deploy-<env>
trigger: issue_comment → created
deploy-development.yml
Deploy ke development, health check, buka issue staging — auto tanpa approval
trigger: repository_dispatch → deploy-development
deploy-staging.yml
Deploy ke staging, health check, buka issue production (jika chain berlanjut)
trigger: repository_dispatch → deploy-staging
deploy-production.yml
Deploy ke production, health check, Slack notification
trigger: repository_dispatch → deploy-production
cleanup.yml
Hapus release branch lama, simpan 2 terakhir
trigger: push tag v*.*.*
.github/approvers.yml
Daftar approver per environment — development: auto, staging: engineer, production: lead
